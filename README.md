# 题目聚集地

## 前言

有一些牛人认为，设计者应该在正式开始做设计之前先花几年时间把所有问题及其解法都学一遍，然后工作中就直接模式匹配。

我们戏称为认证程序员，虽然是戏称，但是也开了一个脑洞。索性建个组织，再建立此仓库来聚集一些典型问题，用这个组织放一些典型问题的典型解法，为可能有一天出现认证程序员做一点贡献。

## 题目

下面的题目有一些输入重新发明轮子的题目，这类题目你不需要写一个完整轮子，只要把原型做出来就可以了，主要学习解决问题的思路，而不是琐碎的细节。

### XUnit

假装这个世界没有单元测试框架，你来写第一个，回到Kent Beck和Eric Gamma在飞机上结对的时刻，重新体验那个历史。

你可以选择：
1. 不使用反射技术，用比较古老的技术，试着实现一个框架，体验在一个蛮荒的世界里如何快速建造起一个可用的框架原型
2. 使用反射技术、Annotation、Lambda表达式等试着打造一个更接近今天的单元测试框架的框架原型


### Diff

用对象模型构造两组数据，两组数据在数值上有不同，在模型概念都是一样的。比如都有Folder和File，Folder下面可以有Folder，也可以有File，搞两棵树，然后比对得到diff。

### Timeline

建造一个版本控制的原型，不需要真的管理文件，只要基于模型能展现出版本节点、时间线即可。

你可以选择：
1. 单一时间轴，可以把某一个旧的历史复原到最新。
2. 存在分支，可以在任何一个时间点上拉出一条分支，可以把当前节点指到任何一个时间点上去。
3. 支持合并，可能需要先试着完成Diff
